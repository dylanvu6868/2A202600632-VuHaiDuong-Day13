How to run / restart this lab from scratch

1. Start the API server (from the repo root, with the virtualenv created and requirements.txt installed):

.venv\Scripts\python.exe -m uvicorn app.main:app --reload

Keep this terminal open — the dashboard needs it running at http://127.0.0.1:8000.

2. (Optional) Reset to a clean slate — stop the server first, then delete the generated log files so validate_logs.py and this dashboard start from zero:

rm data/logs.jsonl data/audit.jsonl

Restart the server (step 1) afterwards — in-memory /metrics counters reset automatically on restart.

3. Generate traffic (in a second terminal):

.venv\Scripts\python.exe scripts/load_test.py
.venv\Scripts\python.exe scripts/load_test.py --concurrency 5

4. (Optional) Inject an incident to see the dashboard/alerts react, then disable it again:

.venv\Scripts\python.exe scripts/inject_incident.py --scenario rag_slow
.venv\Scripts\python.exe scripts/inject_incident.py --scenario rag_slow --disable

5. Open this dashboard at http://127.0.0.1:8000/dashboard — it auto-refreshes every 20s (toggle with the "Pause auto-refresh" button above, or hit "Refresh now").

6. Validate & check alerts:

.venv\Scripts\python.exe scripts/validate_logs.py
.venv\Scripts\python.exe scripts/check_alerts.py

See README.md for the full setup (creating the venv, installing dependencies, .env) and docs/alerts.md for alert details.
